### Hi there 👋

I'm Stephen — freelance Ruby developer & consultant, based in Italy.

I've been writing **Ruby** for over 10 years and love it ❤️. Since [Hacktoberfest '22](https://hacktoberfest.com/) and reading [How To Open Source](https://howtoopensource.dev/) by [Richard Schneeman](https://schneems.com/about), I've been focusing more on OSS. Most of my gems are extracted from production apps I maintain.

Find my gems on [gem.coop](https://beta.gem.coop/@stephen) or [Rubygems](https://rubygems.org/profiles/stevegeek).

## Active projects

- [**vident**](https://github.com/stevegeek/vident) — The base of your design system. Stimulus-powered view components for ViewComponent or Phlex.
- [**typed_operation**](https://github.com/stevegeek/typed_operation) — Command pattern: callable, chainable, traceable, with typed parameters, partial application, currying, and optional authorization.
- [**encoded_id**](https://github.com/stevegeek/encoded_id) — Reversible, human-friendly obfuscated IDs from numerical or hex input. Supports Rails ActiveRecord.
- [**quo**](https://github.com/stevegeek/quo) — Composable query objects for ActiveRecord.
- [**awfy**](https://github.com/stevegeek/awfy) — CLI for running benchmark suites and comparing results over time, across branches, with/without YJIT.
- [**sentiero**](https://github.com/stevegeek/sentiero) — Browser session recording for Ruby. Works with any Rack framework. Privacy-first defaults.
- [**yaml_csp_config**](https://github.com/stevegeek/yaml_csp_config) — Configure Rails content security policy (CSP) in YAML & modify via env vars.
- [**another_api**](https://github.com/stevegeek/another_api) — Rails engine + support gems for building opinionated JSON APIs. Typed schemas, serialization, and a filter/sort query language.
